Study Summary

This project centers on a Pygame simulation where Game orchestrates a fixed frame loop and each Square updates motion using jitter, flee behavior away from larger neighbors, chase behavior toward smaller neighbors, size-based speed normalization, and wall bounce constraints.

Control Flow

run() repeats handle_events(), update(), draw(), then clock.tick(FPS) until running is false.

State Mutation

Square.update mutates vx, vy, x, and y using RANDOM_JITTER, FLEE_RADIUS, CHASE_RADIUS, and target_speed().

Rendering Path

draw() clears the screen, draws rectangles with int coordinates, renders FPS text, and flips the display.

Complexity Signal

Each square scans all squares for flee and chase influence, producing an all-pairs update pattern in main.py.
